# Neko Guess — Design Spec
**Date:** 2026-03-20
**Status:** Approved by user

---

## Overview

Neko Guess is a mobile-first Progressive Web App (PWA) charades-style party game designed for English language learners. One player holds their phone on their forehead in landscape orientation; the screen displays a word or phrase that other players mime or describe. The phone player tilts forward to mark "correct" or backward to "skip/pass."

---

## Goals

- Works on any smartphone via a website URL (no app store required)
- Installable as a PWA ("Add to Home Screen")
- Tilt detection works in both landscape orientations (charge port left or right)
- Word lists appropriate for English learners (A1/A2 level vocabulary)
- Neko Guess branding: yellow/blue color scheme, Fredoka One font, cat mascot

## Screen Flow

```
Home → Countdown (3-2-1) → Game → Results
                                      |
                          ← Play Again (back to Home)
```

### Home Screen
- Neko Guess logo + cat mascot
- Timer selector: 30s / 60s / 90s (default: 60s)
- Large "Start" button
- Category selector (for future expansion; MVP uses all words mixed)

### Countdown Screen
- Full-screen 3… 2… 1… Go!
- On iOS: shows "Allow Motion" permission button before countdown starts
- Gives player time to position phone on forehead

### Game Screen
- Landscape-only enforced via CSS (`orientation: landscape` media query + inline prompt)
- Word displayed in very large, centered text (readable from across a room)
- Timer progress bar across the top
- Visual feedback flash: green for correct, red for skip
- Audio feedback: ding (correct), buzz (skip), tick (last 5s), alarm (time up)
- Audio files: `.mp3` format, generated via Web Audio API (no external files needed)
- Audio context initialised on the iOS motion permission tap to satisfy autoplay policy

### Game Screen — Timer Expiry
- When timer reaches 0: alarm plays, current word freezes on screen for 1 second, then auto-navigates to Results
- Any tilt gesture that occurs during the 1-second freeze is ignored

### Results Screen
- "You got X correct!" headline
- Scrollable list in chronological order (order words were seen): each marked correct ✅ or skipped ⏭
- "Play Again" button → back to Home

## Tilt Detection (`useTilt.ts`)

The app uses the `DeviceOrientationEvent` API.

**Axis:** The `beta` axis (front-to-back pitch, range -180° to 180°) is used for tilt detection. When the phone is held on the forehead in landscape and tilted forward (screen toward floor) or backward (screen toward ceiling), that motion is a rotation around the phone's horizontal axis — this is `beta`. `screen.orientation.angle` is used only to confirm landscape orientation is active, not to switch axes.

**Calibration baseline:**
- When the countdown reaches "Go!", the app captures the current `beta` value as the neutral baseline
- All tilt thresholds are relative to this baseline (not absolute 0°)
- This accounts for variations in how each player holds the phone on their forehead

**Thresholds (relative to baseline):**
- Tilt forward/down past **+25°** from baseline → **Correct**
- Tilt backward/up past **-25°** from baseline → **Skip**
- Neutral zone between -25° and +25° — no action

**Anti-jitter lockout:**
- After a tilt registers, a 1-second lockout prevents double-counting

**Screen orientation lock:**
- `screen.orientation.lock("landscape")` called at game start to prevent OS rotation mid-game
- iOS Safari does not support this API; on iOS, a CSS "please rotate" prompt is the only option

**iOS permission:**
- `DeviceOrientationEvent.requestPermission()` must be called on a direct user tap
- Shown as an "Allow Motion" button on the countdown screen before the 3-2-1 starts
- If permission is denied: show a clear message ("Motion access is needed to play. Please reload and allow motion access.") and disable the Start button
- Audio context is initialised on the same tap to avoid iOS autoplay blocks

## Word Lists

Ported from the existing `head-guess` project. Categories include:
- Animals
- Food
- Sports
- Jobs
- Places
- (additional categories from prior project)

Words are A1/A2 English level — simple, common vocabulary. Lists stored in `src/data/wordLists.ts` as arrays, easy to edit. Minimum 30 words per category. Lists ported from `head-guess` project and reviewed for level-appropriateness.

## PWA Configuration

- `manifest.json`: app name "Neko Guess", theme color yellow, display: standalone, orientation: landscape
- Service worker: caches all assets for offline play
- Icons: 192×192 (standard) and 512×512 with `purpose: "any maskable"` for Android adaptive icons — ported from head-guess assets or generated
- Category selector hidden entirely in MVP (not shown as disabled UI)

## Deployment

1. Push code to GitHub repo `Tyebane/neko-guess`
2. Connect repo to Vercel (one-time setup, free)
3. Vercel auto-deploys on every push to `main`
4. Players visit the Vercel URL and add to home screen
